## 2026-03-18 - Prevent Data Exfiltration to LLMs\n**Vulnerability:** Prompt injection and sensitive data exfiltration risk. The application was passing the entire database row (via `select("*")`) to an external AI model (Gemini) when answering general queries.\n**Learning:** When fetching context for LLM prompts from a database, using `select("*")` blindly exposes all columns, including PII and potentially malicious user-provided strings. This violates the principle of least privilege.\n**Prevention:** Always explicitly define the `select()` statement to only include safe, non-sensitive columns necessary for the prompt context.
